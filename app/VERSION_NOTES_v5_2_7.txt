Macy's AP Accountability System - Beta Version v5.2.7

Changed:
- Removed database/data file location from Dashboard.
- Dashboard health box now keeps only operational status like database OK, last backup, late returns, and errors.
- Database/data file location now displays in System instead.
- System page keeps one Database / Data File section only.
- No duplicate data-location buttons.
- Opening System automatically shows the current data file location in the System output box.

Asset entry improvement:
- Add/Edit Asset now has a guided entry flow with Auto-Fill Name, Save + New, duplicate checks, and type-specific guidance for keys and radios.
- CSV asset import now uses the same cleanup and validation rules as manual asset entry.
- Self-test cleanup now closes the temporary database before removing it, which avoids a Windows file-lock failure.

Final mapping and connection check:
- Added an internal asset type field map so each asset type uses only the fields that belong to it.
- Keys keep controlled key number and clear serial/device tag.
- Radios, scanners, and tablets keep serial/device tag and clear controlled key number.
- Temp badges, items, and other assets keep only common asset fields.
- Add/Edit Asset disables fields that do not apply to the selected type.
- Asset exports now include Export All and Export Type actions from the Assets page.
- CSV bundle exports now include settings and writes headers even when a table is empty.
- Header refresh status now shows local/shared mode, refresh interval, latest refresh time, and active data file name.
- Self-test now checks asset mapping, add/reload, edit/reload, checkout, return, and dashboard count logic.

Manager, asset forms, and Excel upgrade:
- Manager page now has a dedicated backup status band with last backup date, time, status, refresh countdown, and Refresh Now.
- Asset records now support structured type-specific details stored in the database as asset_details.
- Add/Edit Asset now rebuilds its detail fields when the asset type changes.
- Key rings can track set number, ring serial, number of keys, location, use, and each individual key/access description.
- Radios, tablets, temp badges, scanners, items, and other assets have their own matching fields.
- Tablet accessories can be selected and can include a custom Other accessory.
- Validation now checks required type-specific fields, numeric key counts, and duplicate type-specific asset numbers/serials.
- Assets page now includes an Excel export option with an asset-type dropdown.
- Excel export creates a .xlsx workbook organized by asset type.
- Asset import template now includes type-specific columns and every row is generated from the same column map.

Module cleanup:
- Excel workbook creation was moved into macys_ap_export.py.
- Main app now keeps a small write_xlsx wrapper for compatibility while export internals live in the export module.
- This reduces risk when improving Excel output later because export XML logic is no longer mixed into the main UI/database file.
